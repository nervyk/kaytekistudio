# DNS CONFIGURATION REFERENCE

Этот файл содержит шаблон DNS записей для вашего домена.

## После создания сайта на Netlify

Netlify выдаст вам примерно такие данные:

### Для www поддомена:

```
Type:  CNAME
Name:  www.kaytekistudio.ru
Value: XXXXXX.netlify.app (замените на то, что выдаст Netlify)
TTL:   3600 (или 300 для более быстрого обновления)
```

### Для корневого домена (@):

**Опция 1 - ALIAS/ANAME (некоторые регистраторы):**
```
Type:  ALIAS или ANAME
Name:  @ или kaytekistudio.ru
Value: XXXXXX.netlify.app
TTL:   3600
```

**Опция 2 - A Records (если нужны IP адреса):**
Netlify предоставит IP адреса, обычно:
```
Type:  A
Name:  @
Value: 75.2.60.5
TTL:   3600
```

И ещё один A record:
```
Type:  A
Name:  @
Value: 34.200.242.230
TTL:   3600
```

## Популярные регистраторы

### Яндекс.Домены
1. Зайдите в управление доменом
2. Перейдите в "DNS и WHOIS"
3. Нажмите "Добавить запись"
4. Выберите тип CNAME/A и добавьте записи

### Reg.ru
1. Перейдите в "Управление доменом"
2. Нажмите на "Редактировать зону DNS"
3. Добавьте записи типа CNAME

### GoDaddy
1. Перейдите в "DNS Management"
2. Нажмите "Edit" для нужного домена
3. Добавьте CNAME записи

### Timeweb, Regtime и др.
Обычно интерфейс похож - найдите раздел "DNS" или "Управление зоной"

## Проверка DNS распространения

После добавления записей проверьте статус здесь:
- https://www.dnschecker.org/
- https://mxtoolbox.com/

Вводите: `kaytekistudio.ru` и `www.kaytekistudio.ru`

---

## Что делать, если не работает?

1. Подождите 24-48 часов (DNS распространяется)
2. Очистите кэш браузера (Ctrl+Shift+Del)
3. Попробуйте использовать другой браузер
4. Проверьте, что DNS записи добавлены правильно в панели регистратора
5. В Netlify проверьте статус домена в "Site settings"

---

## Важно!

- **TTL значение** - время кэширования DNS. Используйте 300-3600 секунд
- Используйте **только CNAME для www**
- Для корня (@) используйте **ALIAS** или **A records**
- Не забудьте добавить **оба** домена (с www и без www)
