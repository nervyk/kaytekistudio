## 🚀 СЛЕДУЮЩИЕ ШАГИ

### Этап 1️⃣: GitHub (5 минут)
```bash
1. Откройте https://github.com/new
2. Создайте репозиторий "kaytekistudio" (Public)
3. Выполните команды в PowerShell:

cd "d:\KAYTOKI\site"
git remote add origin https://github.com/ВАШ_ЛОГИН/kaytekistudio.git
git branch -M main
git push -u origin main
```

**Результат**: Репозиторий на GitHub  
**URL**: `https://github.com/ВАШ_ЛОГИН/kaytekistudio`

---

### Этап 2️⃣: Netlify (5 минут)
```
1. Перейдите на https://app.netlify.com
2. Нажмите "Connect to Git"
3. Авторизуйтесь через GitHub
4. Выберите репозиторий kaytekistudio
5. Оставьте настройки по умолчанию
6. Нажмите "Deploy site"
```

**Результат**: Сайт на Netlify  
**URL**: `https://XXXXXX.netlify.app` (Netlify выдаст точный адрес)

---

### Этап 3️⃣: Кастомный домен (10 минут)
```
1. В Netlify: Site settings → Domain management
2. Нажмите "Add custom domain"
3. Введите: kaytekistudio.ru
4. Скопируйте DNS записи
5. Зайдите в панель регистратора домена
6. Добавьте DNS записи в управление доменом
7. Подождите 24-48 часов распространения
```

**Типичные DNS записи**:
```
CNAME: www.kaytekistudio.ru → [netlify-domain].netlify.app
A или ALIAS: kaytekistudio.ru → [netlify-ip]
```

## 📞 ТЕХПОДДЕРЖКА

Если что-то не работает:

1. **404 на изображения**
   - Проверьте что папка `assets/images/` создана
   - Проверьте что все файлы скопированы
   - Проверьте пути в HTML (должны быть `./assets/images/`)

2. **Ошибка при гите**
   - Убедитесь что установлен Git
   - Убедитесь что находитесь в правильной папке

3. **DNS не работает**
   - Подождите 24-48 часов
   - Очистите кэш браузера (Ctrl+Shift+Del)
   - Проверьте DNS на dnschecker.org
